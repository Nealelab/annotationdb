static read(path)[source]¶

Read an interval tree from a file.

The File Format

Hail expects an interval file to contain either three or five fields per line in the following formats:

  • contig:start-end
  • contig  start  end (tab-separated)
  • contig  start  end  direction  target (tab-separated)

Note

start and end match positions inclusively, e.g. start <= position <= end. Interval.parse() is exclusive of the end position.

Note

Hail uses the following ordering for contigs: 1-22 sorted numerically, then X, Y, MT, then alphabetically for any contig not matching the standard human chromosomes.

Caution

The interval parser for these files does not support the full range of formats supported by the python parser Interval.parse(). ‘k’, ‘m’, ‘start’, and ‘end’ are all invalid motifs in the contig:start-end format here.

Parameters:path (str) – file name
